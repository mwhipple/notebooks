It can also be invoked from the menu: Help → Describe → Describe

(describe-function FUNCTION)

Display the full documentation of FUNCTION (a symbol).
When called from Lisp, FUNCTION may also be a function object.

See the ‘help-enable-symbol-autoload’ variable for special
handling of autoloaded functions.

  Probably introduced at or before Emacs version 22.1.

*** describe-mode

#+begin_src emacs-lisp
  (describe-function #'describe-mode)
#+end_src

#+RESULTS:
#+begin_example
describe-mode is an autoloaded interactive byte-code-function in
‘help-fns.el’.

It is bound to C-h m, <f1> m and <help> m.
It can also be invoked from the menu: Help → Describe → Describe
Buffer Modes

(describe-mode &optional BUFFER)

Display documentation of current major mode and minor modes.
A brief summary of the minor modes comes first, followed by the
major mode description.  This is followed by detailed
descriptions of the minor modes, each on a separate page.

For this to work correctly for a minor mode, the mode’s indicator
variable (listed in ‘minor-mode-alist’) must also be a function
whose documentation describes the minor mode.

If called from Lisp with a non-nil BUFFER argument, display
documentation for the major and minor modes of that buffer.

When ‘describe-mode-outline’ is non-nil, Outline minor mode
is enabled in the Help buffer.

  Probably introduced at or before Emacs version 29.1.

#+end_example

‘help-fns.el’.

It is bound to C-h v, <f1> v and <help> v.
It can also be invoked from the menu: Help → Describe → Describe
Variable...

(describe-variable VARIABLE &optional BUFFER FRAME)

Display the full documentation of VARIABLE (a symbol).
Returns the documentation as a string, also.
If VARIABLE has a buffer-local value in BUFFER or FRAME
(default to the current buffer and current frame),
it is displayed along with the global value.

  Probably introduced at or before Emacs version 27.1.

** makunbound

#+begin_src emacs-lisp
  (describe-function #'makunbound)
#+end_src

#+RESULTS:
#+begin_example
makunbound is a primitive-function in ‘C source code’.

(makunbound SYMBOL)

Empty out the value cell of SYMBOL, making it void as a variable.
Return SYMBOL.

If a variable is void, trying to evaluate the variable signals a
‘void-variable’ error, instead of returning a value.  For more
details, see Info node ‘(elisp) Void Variables’.

See also ‘fmakunbound’.

#+end_example
