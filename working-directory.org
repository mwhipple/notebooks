#+title: working-directory

The working directory establishes a location context within a
filesystem - relative paths would be resolved...relative to the
working-directory.

/Ideally/ this is little more than a convenience, where the
combination of the working directory and the relative paths could be
replaced by absolute paths, and the working directory is used for
things like more succinct file references or being able to do what
amounts to applying some behavior that is packaged using relative
paths to a conforming file system location.

In other cases some software may assume or require relative paths, and
then those paths and the working-directory become essential rather
than convenient. Sometimes this just reflects deficiencies, though it
may also be in support of security (though it seems as though security
needs would be better addressed by namespacing in modern systems).
