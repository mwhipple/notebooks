#+title: context-argument

A pattern which I make use of widely is the use of a somewhat
expandable context argument. This is a particularly prominent practice
in Go, but generally provides a particularly straightforward way to
allow functions access to additional context as part of their explicit
parameters.

Common alternatives tend to rely on less explicit approaches (such as
reaching into something like a static/global or presumed enclosing
binding environment).
