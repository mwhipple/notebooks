* Setting up a Virtual Environment
:PROPERTIES:
:ID:       A3C4A953-79CA-4160-AB26-A6EDCCD8AC09
:END:

Python can manage local environments with dependencies using its
built-in ~venv~ functionality. Within a notebook this will be
supported through creating the relevant directory, and then
updating blocks to use the python within that directory.

Creating under ~/tmp~ seems like a reasonable practice by default.

** Create the venv

This invokes the host python.

#+name: create-venv
#+begin_src shell :var host_py="python3" :var venv="/tmp/test"
  ${host_py} -m venv ${venv}
#+end_src

#+RESULTS:

#+begin_src shell :var venv="/tmp/test" :results output
  tree ${venv}
#+end_src

** Installing dependencies (w/ pip)

Dependencies can be installed within the venv by invoking ~pip~ with
the venv python.

#+begin_src shell :var venv_py="/tmp/test/bin/python" :results output
  ${venv_py} -m pip install tiktoken
#+end_src

#+begin_src emacs-lisp
  (defun mw/pyvenv-header-args (host_py venv)
    "Produce a value for header args with the provided host_py and venv variabels.
    (insert (string-join
      (list
       (format ":var host_py=\"%s\"" host_py)
       (format ":var venv=\"%s\"" venv)
       (format ":var venv_py=\"%s/bin/python\"" venv))
       (format ":python \"%s\"" host_py)
      " ")))
#+end_src
